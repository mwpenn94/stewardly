# Settings Page Mobile Audit

## Issue: Two-column layout on mobile (390px)
The Settings page shows a side-by-side layout with left nav (SETTINGS menu) and right content panel. On a 390px mobile viewport, both columns are squeezed, making the right panel content (toggle cards) very narrow and hard to read. The text wraps excessively ("Auto-expand actions" becomes multi-line).

## Fix needed:
On mobile, the Settings page should stack vertically — show the nav as a horizontal tab bar or collapsible menu at the top, then full-width content below.
